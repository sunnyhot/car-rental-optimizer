# Changelog

## [Unreleased]

## [0.9.1] - 2026-06-25

### Added
- 候选方案列表新增结果内筛选，可按平台、车型类别、总成本、距离、费用完整度筛选，并支持按门店或车型保留最低价。

### Fixed
- 车型确认搜索改为大半径减量策略：当前位置所在城市只查最近网点，其他城市只查火车站/高铁站旁候选网点，降低平台接口请求量。
- 一嗨车型确认查询的跨城市门店和报价请求改为有上限并发，减少多个城市串行等待导致的 35 秒超时。
- 重新查询会复用同条件下已经成功返回车型的平台结果，只重新调用上次失败或无车型的平台，避免神州成功后一嗨失败时重复查询神州。
- 一嗨登录页首次打开会优先恢复本机保存的 session，不再把有效登录态当成验证码异常数据清掉。
- 一嗨登录页遇到验证码校验异常时不再自动刷新打断操作，改为本地提示并提供手动重置验证状态。
- 神州登录后会把 zuche.com session cookie 保存到本机 Application Support，并在覆盖安装或重启后恢复用于确认页费用接口。
- 神州 API 网关改为移动域名优先，并在 TLS、超时或断连时自动切换备用官方域名，减少安全连接失败导致的整个平台失败。
- 神州登录弹窗改为直接打开官方 H5 登录页并使用移动端 UA，避免桌面首页在 WebView 中空白导致无法登录。
- 神州登录入口切到官方短信验证码 H5 页，并使用完整 iPhone Safari UA、不再注入登录脚本，避免图形验证码请求被本地兼容层或非典型 UA 干扰。
- 神州登录弹窗新增短信登录/密码登录两个官方入口，短信图形验证码异常时可切换到密码页继续登录补费用。
- 首次启动定位授权未完成前不再触发超时重试，并合并并发定位请求，避免系统定位授权弹窗重复出现。
- 神州候选在检测到登录态时会读取官方确认页的基础服务费并计入总成本；未登录时不再用固定费用猜测。
- 神州已返回车源但确认页费用需要登录时，左侧平台状态会主动显示“登录神州补费用”入口，登录完成后可直接重新比较。
- 工作台颜色改为跟随 macOS 明暗外观，修复暗黑模式下标题、日期弹层和状态文字对比度过低的问题。

## [0.9.0] - 2026-06-16

### Added
- 搜索主流程新增查询前条件提示、搜索诊断摘要和平台失败恢复建议。
- 新搜索失败时会保留上次成功候选，并明确标记为历史结果，避免平台临时异常时清空上下文。
- 候选列表、推荐明细和平台对比统一展示报价可信度标签。
- 监控中心新增筛选、健康摘要、attention-first 排序、批量暂停/恢复和当前筛选巡查。
- 价格监控新增连续失败与恢复事件，并展示更完整的趋势摘要和监控匹配说明。
- 菜单快捷键新增重新比较、打开监控中心和立即巡查到期监控。
